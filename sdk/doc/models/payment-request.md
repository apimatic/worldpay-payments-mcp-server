
# Payment Request

*This model accepts additional fields of type unknown.*

## Structure

`PaymentRequest`

## Fields

| Name | Type | Tags | Description |
|  --- | --- | --- | --- |
| `transactionReference` | `string` | Required | A unique reference generated by you that is used to identify a payment throughout its lifecycle.<br><br>**Constraints**: *Minimum Length*: `1`, *Maximum Length*: `64`, *Pattern*: ``^[-A-Za-z0-9_!@#$%()*=.:;?\[\]{}~`/+]*$`` |
| `merchant` | [`Merchant`](../../doc/models/merchant.md) | Required | An object that contains information about the merchant. |
| `instruction` | [`PaymentsInstruction`](../../doc/models/payments-instruction.md) | Required | An object that contains all information related to the payment and API configurations. |
| `channel` | [`Channel1 \| undefined`](../../doc/models/channel-1.md) | Optional | Interaction between the cardholder and you. Supply a value of `ecom` to process an eCommerce authorization. Supply a value of `moto` to process an authorization as a [__Mail Order or Telephone Order__](/products/payments/enable-features/moto) transaction.<br><br>__Note: 3DS authentication cannot be supplied for MOTO payments__. |
| `additionalProperties` | `Record<string, unknown>` | Optional | - |
